# 🏢 WIKISOFT 프로젝트

**퇴직연금 컨설팅을 위한 HR/재무 데이터 자동 검증 시스템**

> 사람이 30분 걸리던 엑셀 검증 작업을 AI가 5분 안에 처리합니다.

---

## 📌 이 프로젝트가 하는 일

퇴직연금 컨설팅 업무에서 고객사로부터 받은 **재직자명부, 급여대장** 같은 엑셀 파일을 검증합니다:

1. **파일 업로드** → 엑셀 파일을 드래그앤드롭
2. **자동 분석** → AI가 컬럼 헤더를 인식하고 표준 필드에 매핑
3. **데이터 검증** → 주민번호 형식, 입사일 논리, 급여 이상치 등 체크
4. **결과 리포트** → 오류/경고를 정리해서 보여줌

**핵심 가치**: 수작업을 줄이고, 실수를 방지하며, 검증 기준을 자동으로 학습합니다.

## 🔮 향후 고려사항

### 1. 워크플로우 자동화 플랫폼이란?

**워크플로우 자동화 플랫폼**은 여러 작업(스크립트, API 호출, 조건 분기 등)을 시각적으로 연결하여 복잡한 비즈니스 프로세스를 자동화하는 도구입니다.

```
예시: Windmill, n8n, Airflow, Temporal, Prefect 등
```

#### AI Agent만 사용 vs 워크플로우 플랫폼 비교

| 구분 | AI Agent만 사용 | 워크플로우 플랫폼 + AI |
|------|----------------|----------------------|
| **장점** | 빠른 개발, 유연한 대화형 처리 | 안정성, 모니터링, 재시도 자동화 |
| **단점** | 실패 시 추적 어려움, 상태 관리 힘듦 | 초기 설정 필요, 학습 곡선 |
| **적합한 경우** | 단순 질의응답, 일회성 작업 | 반복 작업, 멀티스텝 프로세스 |
| **에러 처리** | 직접 구현 필요 | 자동 재시도, 알림, 롤백 |
| **실행 이력** | 로그 직접 관리 | 대시보드에서 전체 조회 |
| **확장성** | 코드 수정 필요 | 드래그앤드롭으로 단계 추가 |

#### 왜 워크플로우 플랫폼을 고려하는가?

1. **신뢰성** - AI Agent가 실패해도 워크플로우가 재시도/알림 처리
2. **가시성** - 누가 언제 무엇을 실행했는지 한눈에 파악
3. **분리** - AI는 "생각"만, 워크플로우는 "실행 관리"만 담당
4. **규정 준수** - 감사 로그 자동 생성 (금융/HR 업무에 중요)

### 2. Windmill 통합 구상

현재는 API를 직접 호출하는 구조지만, 향후 **Windmill** 같은 워크플로우 엔진과 통합을 고려하고 있습니다:

```
현재:   고객 → 파일 업로드 → WIKISOFT3 API → 결과

향후:   고객 → Windmill 워크플로우 → WIKISOFT3 (MCP 서버) → 결과
              ├── 고객 온보딩
              ├── 서류 수집 자동화
              ├── AI 검증 (WIKISOFT3)
              ├── 계리계산
              └── 리포트 생성
```

**기대 효과:**
- 🔐 **보안 강화** - API가 직접 노출되지 않고 Windmill이 게이트웨이 역할
- ⚡ **안정성 향상** - 큐잉, 재시도, 부분 실패 허용
- 📊 **모니터링** - 실행 이력, 감사 로그 자동화
- 🔄 **자동화 확대** - 파일 업로드부터 최종 리포트까지 원스톱

> **참고**: Windmill은 오픈소스 무료 도구로 프로토타입 검증에 적합하며, 상용화 시에는 Temporal, AWS Step Functions 등 엔터프라이즈 도구로 전환 가능합니다.

### 3. MCP (Model Context Protocol) 서버

WIKISOFT3를 **MCP 서버**로 감싸면 AI Agent가 도구로 호출 가능:
- `validate_employee_list` - 재직자명부 검증
- `get_validation_status` - 검증 상태 조회
- `generate_validation_report` - 리포트 생성

**MCP의 역할**: 기존 API를 변경하지 않고 "입구를 하나 더 추가"하는 개념. 웹 UI, REST API, MCP가 동시에 공존 가능합니다.

## 💡 핵심 철학

1. **AI는 가속기, 폴백이 기본** - AI 없어도 동작, 있으면 더 똑똑해짐
2. **UX가 기술보다 중요** - 사용자가 이해하기 쉬운 결과 제공
3. **경고는 차단이 아니라 안내** - 투명하게 보여주고 사용자가 판단
4. **케이스 학습이 미래** - 많이 쓸수록 자동화율 상승
