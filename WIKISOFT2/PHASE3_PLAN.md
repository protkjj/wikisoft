# 🚀 WIKISOFT v3 간단 계획

**날짜**: 2025-12-26

---

## 🎯 목표 (한 줄 요약)
정확도·속도·UX를 동시에 끌어올려 실무 생산성을 극대화.

---

## 핵심 우선순위 (요약)
- 정확도: 별칭/회사별 프로필, 하이브리드 매칭(의미+룰), 수동 매핑 UI
- UX: 진행 자동 저장·되돌리기, 결과 이유 설명, 다국어/접근성
- 성능: 스트리밍 파서, 청크 처리(100k+ 행), 백그라운드 워커
- 보안: 로컬 LLM 옵션, DLP 점검, 감사 로그/역할 기반 접근
- 아키텍처: 파서/스키마 플러그인, 설정-as-코드, 기능 플래그, 텔레메트리
- 연동: Slack/Email 알림, CLI/배치 파이프라인, 오프라인 배포(데스크톱)
- 테스트: 골든 데이터셋, 합성 데이터, 성능 벤치마크, 회귀 스위트

---

## 빠른 승리 (Quick Wins)
- 수동 매핑 UI: 드래그앤드롭 헤더→표준필드 + 신뢰도 표시
- 신뢰도/경고 대시보드: 파일별 매칭률·경고 유형 가시화
- 스트리밍 파서: 진행률 표시 + 부분 결과로 대용량 대응
- 회사 프로필: 고객사별 별칭/관례 저장으로 재사용 정확도 향상

---

## 간단 로드맵 (예시)
- Sprint 1: 수동 매핑 UI, 회사 프로필 저장/불러오기
- Sprint 2: 스트리밍 파서 + 진행률, 대시보드 1차
- Sprint 3: 청크 처리/워커, 텔레메트리·기능 플래그

---

## 수용 기준 (샘플)
- 수동 매핑으로 필수 필드 누락 경고 50% 이상 감소
- 100k+ 행 파일에서 파싱/검증 중 UI 멈춤 없음
- 대시보드에서 파일별 매칭률·경고 Top3가 즉시 조회 가능
